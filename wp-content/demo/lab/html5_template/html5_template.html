Canvas基本详解

使用canvas编程，首先要获取其上下文（context）。接着在上下文中执行动作，最后将这些动作应用到上下文中。可以将canvas的这种编辑方式想象成为数据库事务：开发人员先发起一个事务，然后执行某些操作，最后提交事务。

访问页面的时候，如果浏览器不支持canvas元素，或者不支持HTML5 Canvas API中的某些特性，那么开发人员最好提供一份替代代。例如，开发人员可以通过一张替代图片或者一些说明性的文字告诉访问者，使用最新的浏览器可以获得更佳的浏览效果。代码清单2-2展示了如何在canvas中指定替代文本，当浏览器不支持canvas的时候会显示这些替代内容。除了上面代码中的文本外，同样还可以使用图片，不论是文本还是图片都会在浏览器不支持canvas元素的情况下显示出来。

开发者迫切希望Canvas可以受到广泛支持，因此不断有项目启动来尝试解决老浏览器或者非标准浏览器不支持Canvas的问题。微软已经宣布Internet Explorer 9将支持canvas，因此，所有主流浏览器都支持canvas已经指日可待了。由于各家浏览器对canvas的支持程度有差异，所以最好在使用API之前，先测试一下HTML5 Canvas是否被支持。2.2.1节会讲解怎样通过代码来检测浏览器支持Canvas的情况。

繪製路徑

要使用路徑繪製圖形，我們還需要一些步驟。
beginPath() closePath() stroke() fill()

繪製路徑的第一個步驟是呼叫 beginPath 方法。就本質而言，路徑是以列表形式儲存的一系列子路徑（直線、弧線等），並以此構成一個圖形。每回呼叫 beginPath 方法時，列表便會被清除，我們就可以開始畫新圖形。

第二個步驟是呼叫一個方法，實際繪製路徑。我們很快就會看到。

第三個步驟是呼叫 closePath 方法（這個步驟不是必要的）。這個方法會嘗試以直線連接起點和終點以閉合路徑。如果圖形已閉合或只有一個點，這個函數就什麼也不做。

最後一個步驟是呼叫 stroke 或 fill 方法。呼叫其中一個便會在 canvas 上真正繪出圖形。stroke 用於繪製圖形的框線，fill 則用於繪製填滿的圖形。

附註︰呼叫 fill 方法時，任何未閉合的圖形都將會自動的閉合，也就不需要使用 closePath 方法。

直线：lineTo

lineTo(x, y)

下面的範例繪出兩個三角形，一個實心一個空心（見右圖）。首先呼叫 beginPath 方法開始繪製新的圖形路徑。然後使用 moveTo 方法將起點移到想要的位置。再畫出三角形的兩條邊。你將注意到填滿和只畫邊線這兩個三角形的不同。如前所述，由於將路徑填滿時路徑會自動閉合，而只畫邊線的不會。我們如果去掉空心三角形的 closePath()，就只會畫出兩條直線，而非完整的三角形。

						ctx.lineTo(105,25);

						ctx.lineTo(125,45);
						ctx.lineTo(45,125);
						ctx.closePath();
						ctx.stroke();

套用樣式和色彩

在繪製圖形的章節中，我只使用過直線和填充的預設樣式。在本章中，我們將會探討 canvas 的全部選項，使我們能夠繪製更加吸引人的內容。

色彩

到目前為止，我們只看過繪製內容的方法。如果我們想要給圖形套用色彩，這裡有兩個相當重要的屬性可以使用︰fillStyle 和 strokeStyle。

strokeStyle 是用來設定圖形輪廓的色彩，fillStyle 則是用於填充顏色。color 可以是 CSS 色彩值的字串、漸層物件，或者是圖樣物件。稍後我們會看到漸層和圖樣物件。預設情況下，輪廓線和填充色彩都是設成黑色（CSS 色彩值 #000000）。

fillStyle = color
strokeStyle = color

透明度

除了繪製不透明圖形以外，我們也可以繪製半透明的圖形。只需要設定 globalAlpha 屬性，或是給邊緣線或填充類型指定一個半透明彩色。 globalAlpha = transparency value 這個屬性會給所有繪製在 canvas 上的圖形套用透明值。數值的範圍是從 0.0（完全透明）到 1.0（完全不透明）。這個屬性的預設值是 1.0（完全不透明）。 當我們想要在畫布上繪製許多帶有相同透明度的圖形時，globalAlpha 屬性會很有用。不過我想下面這個選擇會更實用。 strokeStyle 和 fillStyle 屬性接受 CSS 3 色彩值，我們可以使用下面的記法設定帶有透明度的色彩。

// 給輪廓線和填充類型指定帶有透明度的色彩
ctx.strokeStyle = "rgba(255,0,0,0.5)";
ctx.fillStyle = "rgba(255,0,0,0.5)";

陰影

接下来，基于这个上下文执行画线的操作。在代码清单中，调用了三个方法——beginPath、moveTo和lineTo，传入了这条线的起点和终点的坐标。

变换方法——translate函数

路径: 到目前为止，我们已经用到了几个简单的context路径函数。

  1. moveTo(x, y)：不绘制，只是将当前位置移动到新的目标坐标(x,y)。
  2. lineTo(x, y)：不仅将当前位置移动到新的目标坐标(x,y)，而且在两个坐标之间画一条直线。
  3. stroke(): 填充路径。
  4. strokeStyle（通过strokeStyle属性改变了线条的颜色。在这个例子中，我们使用了CSS值来设置颜色）
  5. fill(): 填充区域。
  6. fillStyle: 填充区域。
  7. closePath:。这个函数的行为同lineTo很像，唯一的差别在于closePath会将路径的起始坐标自动作为目标坐标。closePath还会通知canvas当前绘制的图形已经闭合或者形成了完全封闭的区域，这对将来的填充和描边都非常有用。
  8. lineWidth（将线条宽度加粗到4像素。）
  9. lineJoin（lineJoin属性设置为round，这是修改当前形状中线段的连接方式，让拐角变得更圆滑；也可以把lineJoin属性设置成bevel或者miter（相应的context.miterLimit值也需要调整）来变换拐角样式。）
  10. 还有一个没有用到的属性——lineCap，可以把它的值设置为butt、square或者round，以此来指定线条末端的样式。哦，示例中的线是闭合的，没有端点。图2-7就是我们加工过的树冠，与之前扁平的黑线相比，现在是一条更粗、更平滑的棕色线条。
  11. context.fillRect(-5, -50, 10, 50);画矩形
    strokeRect和clearRect。strokeRect的作用是基于给出的位置和坐标画出矩形的轮廓
    clearRect的作用是清除矩形区域内的所有内容并将它恢复到初始状态，即透明色。
  12. quadraticCurveTo(170, -50, 260, -190);用最简单的曲线函数——二次曲线，用来绘制曲线(HTML5 Canvas API的其他曲线功能还涉及bezierCurveTo、arcTo和arc函数。这些函数通过多种控制点（如半径、角度等）让曲线更具可塑性)
  13. drawImage(bark, -5, -50, 10, 50); bark是图片变量，在canvas中插入图片:图片增加了canvas操作的复杂度：必须等到图片完全加载后才能对其进行操作。浏览器通常会在页面脚本执行的同时异步加载图片。如果试图在图片未完全加载之前就将其呈现到canvas上，那么canvas将不会显示任何图片。因此，开发人员要特别注意，在呈现之前，应确保图片已经加载完毕。
  14. context . restore() //弹出堆最上面保存的绘图状态
  15. context . save() //在绘图状态堆上保存当前绘图状态
  16. context . rotate(angle) //按给定的弧度旋转,按顺时针旋转
  17. context . scale(x, y) //按给定的缩放倍率缩放，1.0,为不变
  18. context . fillStyle [ = value ] //返回填充形状的当前风格，能被设置以用来改变当前的填充风格，其值可以是CSS颜色字串，也可以是CanvasGradient或者CanvasPattern对象，非法的值将被忽略。
  19. context . strokeStyle [ = value ] //返回当前描绘形状的风格，能被设置，其值同上。

简而言之，上面两个函数的区别在于：moveTo就像是提起画笔，移动到新位置，而lineTo告诉canvas用画笔从纸上的旧坐标画条直线到新坐标。不过，再次提醒一下，不管调用它们哪一个，都不会真正画出图形，因为我们还没有调用stroke或者fill函数。目前，我们只是在定义路径的位置，以便后面绘制时使用。

《描述文档用标签》简介

cite是文档标签，值为链接;指向另一个文档的 URL，该文档解释文本插入的原因。
del是文档删除，值为链接
datetime 定义文本插入的日期和时间。
ins 标签定义文档的其余部分之外的插入文本。
details 标签用于描述文档或文档某个部分的细节。details内容点击summary内容展开
summary 标签包含 details 元素的标题，"details" 元素用于描述有关文档或文档片段的详细信息。
